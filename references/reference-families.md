# Reference Families

Use these as design vocabulary. Do not clone brands, copy proprietary assets, or imply affiliation. Pick the family that matches the surface and user job.

## Operational SaaS

Examples: Linear, Vercel dashboard, GitHub settings, Supabase dashboard, Stripe dashboard.

Use for:

- Workspaces, issue trackers, developer tools, CRM, admin, analytics, configuration, internal apps.

Traits:

- Dense but calm.
- Strong nav hierarchy.
- Low decoration.
- Tables, lists, filters, panels, and command surfaces.
- Hairline borders, restrained color, precise spacing.
- Actions near objects.

Avoid:

- Large marketing sections.
- Decorative cards.
- Purple glow unless it is an explicit brand token.
- Fake product screenshots.

## Developer Tools

Examples: Vercel, Raycast, Resend, Supabase, Mintlify, GitHub, Stripe docs.

Use for:

- APIs, CLIs, SDKs, docs, deployment tools, integrations, AI agent tools.

Traits:

- Code-forward layouts.
- Monospace accents.
- Clear examples.
- Copyable commands.
- Tight nav and keyboard-friendly flows.
- Product screenshots or real terminal output.

Avoid:

- Stock illustrations instead of product proof.
- Decorative terminals with fake commands.
- Dense gradients that reduce code readability.

## Marketing Landing

Examples: Stripe, Linear, Apple, Framer, Webflow, Wise.

Use for:

- Public pages, campaigns, startup homepages, product launches, conversion surfaces.

Traits:

- First viewport communicates category and offer.
- Visual proof appears early.
- Strong section rhythm.
- Clear CTA hierarchy.
- Media, product shots, diagrams, or real objects carry the page.

Avoid:

- Generic "transform your workflow" copy.
- Abstract mesh backgrounds as the main proof.
- Invented testimonials.
- Overused three-card feature rows.

## Editor And Creator Tools

Examples: Figma, Framer, Webflow, Notion, Canva, VS Code.

Use for:

- Builders, editors, CMS, creative tools, writing tools, canvas workflows.

Traits:

- Canvas first.
- Toolbars, inspectors, layers, preview, selection state.
- Compact controls.
- Clear persistence.
- Strong object affordances.

Avoid:

- Marketing chrome inside the editor.
- Cards around the canvas.
- Controls that move during interaction.

## Admin Data Tools

Examples: Shopify admin, Airtable, Retool, Metabase, HubSpot CRM.

Use for:

- Internal operations, record management, data tables, workflow queues, support panels.

Traits:

- Dense tables.
- Bulk actions.
- Saved views and filters.
- Status badges with semantic meaning.
- Inline editing where appropriate.
- Clear audit and ownership fields.

Avoid:

- Dashboard-only pages when the workflow needs records.
- Big decorative stat cards.
- Empty states without a next action.

## Fintech And Trust

Examples: Stripe, Wise, Coinbase, Mastercard, Mercury, Ramp.

Use for:

- Payments, billing, pricing, banking, security, compliance, finance analytics.

Traits:

- Precise data formatting.
- Conservative hierarchy.
- Strong contrast.
- Clear risk and recovery language.
- Trust signals close to sensitive actions.
- Charts and tables with labels and units.

Avoid:

- Playful motion around money movement.
- Ambiguous success or pending states.
- Low-contrast numbers.
- Red and green as the only meaning.

## Consumer And Media

Examples: Apple, Nike, Airbnb, Spotify, Pinterest, WIRED.

Use for:

- Consumer launches, media, portfolios, lifestyle, editorial, brand-forward pages.

Traits:

- Real imagery leads.
- Product or object is inspectable.
- More expressive composition is allowed.
- Fewer UI controls.
- Typography and media carry hierarchy.

Avoid:

- SaaS dashboards as decoration.
- Stock-like dark blurred imagery.
- Dense admin controls.

## Choosing A Family

Prefer a specific reference point over generic adjectives. "A technical handout for research engineers" is stronger than "clean and modern". "Linear settings density with Stripe docs clarity" is stronger than "premium SaaS".

When the user names a reference:

- State which traits will be borrowed.
- State what will not be copied.
- Map those traits into the Visual Contract.

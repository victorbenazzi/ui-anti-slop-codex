# Contributing

English first. Portuguese follows.

## English

Contributions are welcome when they make the skill more reliable for real Codex UI work.

Good contributions:

1. Add surface contracts that prevent common UI failure modes.
2. Improve verification guidance.
3. Clarify `DESIGN.md` usage.
4. Add better prompt examples.
5. Reduce ambiguity without making the skill too long.

Before opening a pull request:

1. Keep `SKILL.md` concise.
2. Put detailed guidance in `references/`.
3. Avoid brand copying. References are vocabulary, not permission.
4. Do not add generated screenshots, binaries, or unrelated docs.
5. Validate the skill frontmatter if you have the Codex skill validator available.

## Português

Contribuições são bem-vindas quando tornam a skill mais confiável para trabalho real de UI no Codex.

Boas contribuições:

1. Adicionam contratos de superfície que evitam falhas comuns de UI.
2. Melhoram o guia de verificação visual.
3. Esclarecem o uso de `DESIGN.md`.
4. Adicionam exemplos de prompt melhores.
5. Reduzem ambiguidade sem deixar a skill longa demais.

Antes de abrir um pull request:

1. Mantenha `SKILL.md` conciso.
2. Coloque detalhes em `references/`.
3. Não copie marcas. Referências são vocabulário, não permissão.
4. Não adicione screenshots gerados, binários ou documentação sem relação.
